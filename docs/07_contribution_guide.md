# 07 — Contribution Guide

> **How to contribute new knowledge, artifacts, and improvements to IKIS.**

---

## 1. Who Can Contribute

Contributors include:
- SSS team members
- AI agents (following AI_OPERATING_MANUAL.md)
- External collaborators (by invitation)

## 2. Contribution Workflow

```
1. Identify need → 2. Select template → 3. Create artifact →
4. Add metadata → 5. Validate locally → 6. Submit PR →
7. Review → 8. Merge
```

## 3. Commit Message Format

```
[type]([scope]): [description]
```

### Types

| Type | Use For |
|------|---------|
| `knowledge` | New or updated knowledge entries |
| `decision` | New or updated decision records |
| `spec` | New or updated specifications |
| `template` | New or updated templates |
| `governance` | New or updated governance artifacts |
| `automation` | New or updated automation scripts |
| `docs` | Documentation improvements |

### Scopes

| Scope | Description |
|-------|-------------|
| `sss-domain` | SSS business domain artifacts |
| `pillar-a` | Pillar A: SSS Firm (SSS Firm) |
| `pillar-b` | Pillar B: Afro-Indian-American Strategic Minerals and Capital Corridor (Afro-Indian-American Strategic Trade Alliance (Afro-Indian-American Strategic Trade Alliance (AISTA)) Corridor) (Afro-Indian-American Strategic Minerals and Capital Corridor (Afro-Indian-American Strategic Trade Alliance (Afro-Indian-American Strategic Trade Alliance (AISTA)) Corridor)) |
| `pillar-c` | Pillar C: Corleone Enterprise Holdings (Corleone) |
| `automation` | Automation and tooling |
| `governance` | Governance and process |
| `ai-agents` | AI agent assets |

## 4. Quality Checklist

Before submitting, verify:

- [ ] Metadata block is complete
- [ ] Template structure is followed
- [ ] All factual claims have sources
- [ ] Confidence levels are assigned
- [ ] Legal gating rules applied (SSS materials)
- [ ] Cross-references are accurate
- [ ] Local validation passes
